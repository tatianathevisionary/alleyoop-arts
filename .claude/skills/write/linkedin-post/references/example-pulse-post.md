## Draft output

```
At work this past quarter, I kept noticing the same shape: every product team I
talked to was building "a smarter search." Different surfaces, same pattern.

Same pattern.

Last week at Hack Days 39, a small team I got to build with — @Moujtaba and the
build crew — flipped the question. Not "what if search were smarter?" but
"what if the agent moved before the search?"

I saw the idea months ago in a conversation with a colleague. I didn't sit with
it. Last week I finally did.

**What if Shop Agent knew what you wanted before you asked?**

At the support desk, when I'm investigating a merchant ticket, the signals are
all there before the question gets typed — abandoned cart, return pattern,
inventory state. A specialist reads them in seconds. The buyer types the
question anyway, because the system is built around the question, not the signal.

At home, my kid asks me what's for dinner at 5:47pm. By 5:47pm, dinner is
already decided. The question is a formality. The signal — what I pulled out
of the freezer at 4 — was the real conversation.

Pulse, the prototype we built in 3 days, moves the agent from reactive to
proactive. Reads signals. Acts. Same surface, different posture.

The thing about cascades: the click you keep deferring is usually the one
that flips the shape.

What's the question your system is built around that should have already
been answered?

(Thanks Moujtaba and the HD39 build team — the question matters more than
the answer.)
```
