# Shopify style cheatsheet

Quick reference for the most common style decisions. Full guidance lives at [Polaris product content guidelines](https://polaris.shopify.com/content/product-content). See [Vault 2599 — Writing Principles](https://vault.shopify.io/docs/get-to-know-shopify/190-Shopify-s-Culture/2599-Writing-Principles) for the canonical source.

## Spelling

| Wrong | Right |
|---|---|
| e-commerce, E-commerce, eCommerce | **ecommerce** |
| Internet | **internet** |
| "realise", "organisation", "colour" | **"realize", "organization", "color"** (American English) |
| "utilize" | **"use"** (simpler wins) |

## Capitalization

- **Sentence case for titles.** "Like this one here" — not "Like This One Here."
- Product names stay capitalized: **Shop Pay**, **Sidekick**, **Shopify Plus**.
- Roles are lowercase unless part of a title: "the senior lead" (general) vs. "Senior Lead, Enterprise Support" (formal title).

## Punctuation

- **Oxford comma.** "lions, tigers, and bears" — not "lions, tigers and bears."
- **Single space between sentences.** Never two.
- **Em dash** (`—`) sparingly — two max per page. If every sentence has one, they become tics.
- **Straight quotes** (`"`, `'`) not curly (`"`, `'`) in technical contexts. Match the surrounding context in prose.

## Lists

- Bullets end without periods if they're fragments. With periods if they're sentences.
- Don't mix both in the same list.
- Parallel structure: every bullet starts with the same part of speech (verb, noun phrase, etc.).

## Acronyms

Spell out on first reference, then abbreviate:
- "First Contact Resolution (FCR)"
- "Account Executive (AE)"
- "Customer Success Manager (CSM)"

Check [Rest-in-Peace Acronyms](https://vault.shopify.io/docs/shopifolk-essentials/196-Policies---Guidelines/Guidelines/1456-Rest-in-Peace-Acronyms) for acronyms to avoid entirely.

Never use an acronym in the **first paragraph** of an exec brief — spell out or cut.

## Phrases to cut

| Cut this | Why |
|---|---|
| "I hope this finds you well" | Bureaucratic opener. Delete on sight. |
| "Circling back" / "Touching base" | Say the actual thing. |
| "As I mentioned above" | If the reader got it, redundant. If they didn't, repeating won't help. |
| "Per my last email" | Passive-aggressive. Just re-state the ask. |
| "Thoughts?" | Ask a specific question instead. |
| "At Shopify" | Ideal count: 0. 1 max. Never more. |
| "Going forward" | Redundant. "We will" already implies future. |
| "In terms of" | Usually deletable. |
| "Leverage" (as a verb) | Say "use." |
| "Socialize" | Say "share" or "discuss." |
| "Align on" | Say "agree on" or "decide." |
| "Directionally" | Say "roughly" or cut. |
| "Synergy", "holistic", "thought leadership", "world-class" | Consulting slide language. |

## Active voice vs passive voice

**Default to active.**

| Passive | Active |
|---|---|
| "It was decided that the idea should be presented." | "They decided to present the idea." |
| "The ticket was handled by Dana." | "Dana handled the ticket." |
| "Mistakes were made." | "We made mistakes." (Or: "Who made which?") |

**Use passive only when** the object matters more than the subject:
- "Investment Plans are written each quarter." (the plans are the point, not the writers)
- "The API was deprecated in 2023." (the deprecation fact matters, not who decided)

## Positive vs negative framing

**Describe what happened, not what didn't.**

| Negative | Positive |
|---|---|
| "The specialist didn't escalate in time." | "The specialist escalated after the SLA window." |
| "We don't have a fix yet." | "We're investigating — will update by <time>." |
| "This isn't a priority." | "We're focused on X this quarter." |

## Hedging — when it's fair vs. weasel

**Fair hedge** (source your uncertainty):
- "Per Beacon team's manual check of 18 tickets, routing matches design."
- "Based on the 30-day sample — trend could shift with more data."

**Weasel hedge** (cut these):
- "Maybe we should consider possibly looking into…"
- "It might be worth potentially evaluating…"
- "If we want to, we could directionally…"

## Links

Shopify is high-context. Link instead of re-explaining. Links are the killer feature of the web — use them more than you think.

- Link Slack threads rather than re-summarizing.
- Link dashboard rather than pasting screenshots.
- Link Vault team page rather than listing members.
- Link GitHub PR rather than describing the code change.

## Sentence length — the Gary Provost test

Vary it. Read your draft aloud. If five consecutive sentences are the same length, rewrite one to be shorter and one to be longer. See the [Gary Provost passage in writing-principles SKILL.md](../SKILL.md) for the canonical example.
